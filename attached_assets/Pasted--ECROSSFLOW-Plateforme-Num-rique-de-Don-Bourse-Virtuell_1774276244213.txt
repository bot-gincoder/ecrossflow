# 🌐 ECROSSFLOW
### Plateforme Numérique de Don & Bourse Virtuelle integré- sytem Pyramidale

---

## 0. VISION GLOBALE DU PRODUIT

Ecrossflow est une plateforme numérique de don communautaire à structure pyramidale inspirée du modèle « Gift of Legacy ». Elle combine :
- Un système de boards progressifs (F → S) avec rôles internes
- Une bourse virtuelle multi-devises par utilisateur
- Un système de gestion de fonds (dépôt, retrait, paiement de boards)
- Un historique de transactions complet et des rapports détaillés
- Un système de parrainage et de bonus
- Un système de numérotation de comptes validés
- Une interface complète : multi-langue, multi-thème, mobile-first, animations riches

Stack technique recommandé :
- **Frontend** : Next.js 14 (App Router) + TypeScript + Tailwind CSS + Framer Motion
- **Backend / BDD** : Supabase (Auth, PostgreSQL, Realtime, Storage, Edge Functions)
- **Paiements** : Stripe (carte/bank/PayPal), CoinGate ou NOWPayments (crypto), intégration manuelle Moncash & Natcash (API ou webhook SMS/USSD)
- **Auth** : Supabase Auth + Google OAuth + OTP email/SMS (Twilio ou Supabase built-in)
- **I18n** : next-intl ou i18next (FR, EN, ES, HT — créole haïtien)
- **Thèmes** : next-themes (Light, Dark, Midnight Blue, Gold/Luxury)
- **Devises** : API taux de change (Open Exchange Rates ou Fixer.io) + HTG fixe ($1 = 140 gourdes)

## 4. SYSTÈME D'AUTHENTIFICATION — DÉTAIL COMPLET

### 4.1 Inscription

**Champs du formulaire :**
```
- Prénom (requis)
- Nom (requis)
- Username (unique, minuscules, sans espaces, 3-20 chars, vérification temps réel via debounce 500ms)
- Email (requis, validation format)
- Téléphone (optionnel, format international E.164)
- Mot de passe (min 8 chars, avec indicateur de force visuel : faible/moyen/fort)
- Confirmer mot de passe
- Code de parrainage (OBLIGATOIRE — champ avec validation en temps réel, affiche nom du parrain si valide)
- Case à cocher CGU
```

**Google OAuth :**
- Bouton "Continuer avec Google" (OAuth Supabase)
- Après OAuth : si nouvel utilisateur → modal pour entrer le code de parrainage (obligatoire)
- Le username est pré-rempli depuis le nom Google (modifiable)

**Flux post-inscription :**
1. Email de vérification envoyé → page "Vérifiez votre email"
2. Clic sur lien → retour app → page de vérification OTP optionnelle (SMS si téléphone fourni)
3. Compte créé en statut "PENDING" (en attente d'activation)
4. Redirection vers onboarding

**Page Onboarding (après première connexion) :**
- Étape 1 : Bienvenue + explication du statut "En attente d'activation"
- Étape 2 : Payer $1 pour activer (sélection méthode de paiement)
- Étape 3 : Inviter 2 amis (affichage lien de parrainage + code personnel, boutons partage WhatsApp/Telegram/Copier)
- Étape 4 (déblocage automatique) : Compte numéroté et activé → notification + confettis

### 4.2 Connexion

**Formulaire :**
```
- Email OU Username
- Mot de passe
- Case "Se souvenir de moi"
```

**Après soumission :**
- Si 2FA activé → choix : code par email OU SMS → saisie OTP 6 chiffres (timer 60s + renvoi)

**Google OAuth connexion :**
- Même bouton → si compte existant → connexion directe → OTP si 2FA activé

**Sécurité :**
- Rate limiting : 5 tentatives → blocage 15 minutes (côté Supabase Edge Function)
- Détection connexion depuis nouveau device → email d'alerte automatique
- Sessions JWT avec refresh token (30 jours si "Se souvenir de moi", 24h sinon)

## 5. SYSTÈME DE BOARDS — LOGIQUE COMPLÈTE

### 5.1 Structure des 7 Boards

| Board | Frais Entrée | Multiplicateur | Gain Total Ranker | Déduit (Board Suivant) | Retirable |
|-------|-------------|----------------|-------------------|------------------------|-----------|
| F     | $2          | ×8             | $16               | $8                     | $6        |
| E     | $8          | ×8             | $64               | $32                    | $24       |
| D     | $32         | ×8             | $256              | $128                   | $96       |
| C     | $128        | ×8             | $1,024            | $512                   | $384      |
| B     | $512        | ×8             | $4,096            | $2,048                 | $1,536    |
| A     | $2,048      | ×8             | $16,384           | $8,192                 | $6,143    |
| S     | $8,192      | ×8             | $65,536           | –                      | ~$50,000  |

**Note Board A :** Fee d'entrée = $2,050 (dont $2 de frais plateforme)
**Note Board S :** Fee = $15,000 ; gain total = $65,536 ; après déductions le ranker sort avec ~$50,000

### 5.2 Les 4 Rôles dans chaque Board

```
STARTER → CHALLENGER → LEADER → RANKER
```

**Starter :**
- Paie $1 d'activation (une seule fois, à l'inscription)
- Entre dans le board F comme Starter
- Paie le fee du board au Ranker actuel (via bourse virtuelle)
- Les fonds de 7 starters (sur 8) vont directement au Ranker
- Le 8ème paiement couvre les frais système / prochain board

**Challenger :**
- Devient Challenger après avoir recruté/parrainé suffisamment
- Statut intermédiaire de progression

**Leader :**
- Gère une branche de la pyramide
- Peut voir ses sous-membres

**Ranker :**
- Position de tête du board
- Reçoit les paiements des 7 Starters
- Quand le board est completé (8 personnes payées) → monte automatiquement au board suivant
- Recoit le gain, dont : une partie va vers le board suivant, le reste est retirable

### 5.3 Disposition Pyramidale Visuelle

Pour chaque board, afficher un arbre pyramidal :
```
         [RANKER]
        /        \
   [LEADER]    [LEADER]
   /    \       /    \
[CH]  [CH]  [CH]  [CH]
/\    /\    /\    /\
S  S  S S  S S  S S
```
8 Starters → 4 Challengers → 2 Leaders → 1 Ranker

Animation : Quand un nœud est rempli, il s'illumine (pulse vert). Quand le board est complet → animation de célébration (confettis + modal de félicitations).

### 5.4 Initialisation Admin

- Le tout premier compte admin est automatiquement placé en position **Ranker du Board F**
- Quand un nouveau Ranker arrive en Board F → l'admin monte en Board E
- Ce système se continue automatiquement

### 5.5 Flux de Paiement Board

1. Utilisateur clique "Payer le board [X]"
2. Vérification : fonds suffisants dans bourse virtuelle
3. Déduction immédiate du montant (fee du board)
4. Envoi automatique au Ranker actuel (7/8 des fonds)
5. Le 8ème paiement déclenche la complétion du board
6. Animation de célébration côté Ranker + côté board
7. Ranker monte automatiquement, fonds alloués selon règles
8. Notification push / in-app à tous les membres du board

### 6.2 Dépôt de Fonds

**Flux :**
1. User clique "Déposer"
2. Choisit le montant (champ + suggestions rapides : $10 / $50 / $100 / $500)
3. Choisit la devise d'entrée
4. Choisit la méthode de paiement :

**Méthodes de dépôt :**

a) **Moncash (Haiti)** :
   - Affichage du numéro Moncash de la plateforme
   - Instructions : "Envoyez [montant HTG] au [numéro], puis entrez votre référence"
   - Upload optionnel de screenshot comme preuve
   - Statut : "En attente de validation" (admin valide manuellement)
   - Taux : $1 = 140 HTG (affiché clairement)

b) **Natcash (Haiti)** :
   - Même processus que Moncash

c) **Carte Débit/Crédit** :
   - Intégration Stripe Elements (formulaire carte sécurisé PCI-DSS)
   - Visa, Mastercard, American Express
   - Validation 3D Secure automatique

d) **PayPal** :
   - Redirection vers PayPal SDK
   - Retour automatique après paiement

e) **Virement bancaire** :
   - Affichage des coordonnées bancaires de la plateforme
   - Référence unique générée pour traçabilité
   - Délai : 1-3 jours ouvrables

f) **Crypto** :
   - Bitcoin (BTC), USDT (TRC20/ERC20), ETH
   - Affichage adresse wallet QR code
   - Détection automatique de transaction via webhook NOWPayments/CoinGate
   - Confirmation après X blocs selon la crypto

**Après dépôt validé :**
- Notification in-app + email + SMS optionnel
- Crédit immédiat dans bourse virtuelle
- Entrée dans historique avec tous les détails

### 6.3 Retrait de Fonds

**Conditions :**
- Montant minimum : $5
- Uniquement sur le "Solde disponible" (retirable)
- KYC requis au-delà de $500 (upload pièce d'identité via Supabase Storage)

**Flux :**
1. User clique "Retirer"
2. Saisit le montant
3. Choisit la méthode de retrait (mêmes méthodes que dépôt)
4. Confirmation avec code OTP envoyé par email/SMS
5. Retrait créé en statut "En traitement"
6. Admin valide (Moncash/Natcash) ou automatique (Stripe/Crypto)
7. Notification confirmation

**Délais affichés clairement :**
- Moncash/Natcash : 24-48h (jours ouvrables)
- Carte/Bank : 3-5 jours
- Crypto : 1-24h
- PayPal : Immédiat à 2h

### 6.4 Conversion de Devises

- Interface simple : [montant] [devise source] → [devise cible]
- Taux affiché en temps réel (avec horodatage du taux)
- Taux de conversion appliqué : taux du marché + 1% de frais plateforme
- Confirmation avant exécution
- Conversion instantanée dans la bourse virtuelle

## 8. SYSTÈME PYRAMIDAL — LOGIQUE BASE DE DONNÉES

### 8.1 Tables Supabase Principales

```sql
-- Utilisateurs
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  account_number INTEGER UNIQUE,  -- assigné à la validation
  first_name VARCHAR(100) NOT NULL,
  last_name VARCHAR(100) NOT NULL,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  phone VARCHAR(20),
  avatar_url TEXT,
  referral_code VARCHAR(20) UNIQUE NOT NULL,  -- généré à l'inscription
  referred_by UUID REFERENCES users(id),
  status ENUM('PENDING', 'ACTIVE', 'SUSPENDED') DEFAULT 'PENDING',
  role ENUM('USER', 'ADMIN') DEFAULT 'USER',
  kyc_status ENUM('NONE', 'PENDING', 'APPROVED', 'REJECTED') DEFAULT 'NONE',
  preferred_language VARCHAR(5) DEFAULT 'fr',
  preferred_currency VARCHAR(10) DEFAULT 'USD',
  preferred_theme VARCHAR(20) DEFAULT 'light',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  activated_at TIMESTAMPTZ,
  CONSTRAINT referral_code_format CHECK (referral_code ~ '^[A-Z0-9]{8}$')
);

-- Parrainage (pour tracker l'arbre)
CREATE TABLE referrals (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  referrer_id UUID REFERENCES users(id),
  referred_id UUID REFERENCES users(id),
  bonus_paid BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Boards (configuration)
CREATE TABLE boards (
  id VARCHAR(5) PRIMARY KEY,  -- 'F','E','D','C','B','A','S'
  rank_order INTEGER,
  entry_fee DECIMAL(10,2),
  multiplier INTEGER DEFAULT 8,
  total_gain DECIMAL(10,2),
  next_board_deduction DECIMAL(10,2),
  withdrawable DECIMAL(10,2),
  description TEXT,
  color_theme VARCHAR(50)
);

-- Instances de board (une instance = un groupe de 8 personnes)
CREATE TABLE board_instances (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  board_id VARCHAR(5) REFERENCES boards(id),
  instance_number INTEGER,
  ranker_id UUID REFERENCES users(id),
  status ENUM('WAITING', 'ACTIVE', 'COMPLETED') DEFAULT 'WAITING',
  slots_filled INTEGER DEFAULT 0,
  total_collected DECIMAL(10,2) DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

-- Participation aux boards
CREATE TABLE board_participants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  board_instance_id UUID REFERENCES board_instances(id),
  user_id UUID REFERENCES users(id),
  role ENUM('STARTER', 'CHALLENGER', 'LEADER', 'RANKER'),
  position INTEGER,  -- 1-8 pour les starters
  amount_paid DECIMAL(10,2),
  paid_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()

CREATE TABLE wallets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID UNIQUE REFERENCES users(id),
  balance_usd DECIMAL(18,2) DEFAULT 0,
  balance_pending DECIMAL(18,2) DEFAULT 0,
  balance_reserved DECIMAL(18,2) DEFAULT 0,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Transactions
CREATE TABLE transactions (
  -- (voir section 7.1 pour le schéma complet)
);

-- Bonus
CREATE TABLE bonuses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  type ENUM('REFERRAL_3', 'BOARD_COMPLETION', 'MILESTONE'),

  status ENUM('PENDING', 'PAID'),
```

### 8.2 Triggers et Edge Functions Supabase

**Trigger : Numérotation automatique des comptes**
```sql
-- Quand les 2 conditions sont remplies (paiement $1 + 2 filleuls)
-- → account_number est assigné par ordre d'activation (1, 2, 3...)
-- Le premier compte admin a account_number = 0 (ou réservé)
```

**Edge Function : board_payment_processor**
- Appelée à chaque paiement de board
- Vérifie le solde, déduit, envoie au Ranker
- Si 8ème paiement → déclenche board_completion_handler

**Edge Function : board_completion_handler**
- Crée notification pour tous les membres
- Fait monter le Ranker au board suivant
- Alloue les fonds (retirable + réservé pour prochain board)
- Crée nouvelle instance de board

**Realtime Subscriptions :**
- Board instances : mise à jour en temps réel de la pyramide
- Wallet balance : mise à jour temps réel du solde
- Notifications : push temps réel

## 9. SYSTÈME DE PARRAINAGE & BONUS

### 9.1 Lien et Code de Parrainage

Chaque utilisateur reçoit :
- **Code unique** : 8 caractères alphanum majuscules (ex: `ECF-3X9KM2`)
- **Lien unique** : `https://ecrossflow.com/join?ref=3X9KM2`
- **QR Code** : génératable et téléchargeable

**Page Parrainage (/referrals) :**
- Affichage du lien + code avec bouton "Copier"
- Boutons partage direct : WhatsApp, Telegram, Facebook, Twitter, Email, SMS
- QR Code (généré client-side avec qrcode.js)
- Compteur filleuls : Total | Actifs | En attente
- Tableau des filleuls (pseudo, date d'inscription, statut)
- Barre de progression vers bonus (0/2 pour activation, 0/3 pour bonus)

### 9.2 Règles de Parrainage

**Activation de compte :**
- User doit parrainer 2 personnes qui s'inscrivent ET activent leur compte
- Ou payer $1 + avoir 2 filleuls inscrits (ils n'ont pas besoin d'être actifs)
- Quand les 2 conditions remplies → compte validé → numéro assigné

**Bonus de Recrutement (3 filleuls actifs) :**
- Si un utilisateur recrute 3 filleuls qui activent leur compte
- → Bonus automatique crédité dans sa bourse virtuelle
- Montant du bonus = à définir (ex: $5 ou % du fee du board actuel)
- Notification immédiate : "🎉 Vous avez recruté 3 membres ! Bonus de $X crédité"

**Multi-niveaux (optionnel, à configurer) :**
- Niveau 1 (filleuls directs) : bonus X%
- Niveau 2 (filleuls de filleuls) : bonus Y%
- Configurable dans admin settings

## 14. SÉCURITÉ — CHECKLIST COMPLÈTE

- [ ] Row Level Security (RLS) activé sur toutes les tables Supabase
- [ ] Chaque user ne voit que ses propres données
- [ ] Admin bypass RLS avec service_role key (côté Edge Functions seulement)
- [ ] Validation de toutes les entrées (server-side via Edge Functions)
- [ ] Rate limiting sur auth, paiements, OTP
- [ ] HTTPS uniquement (Vercel auto)
- [ ] Variables d'environnement : jamais de clé côté client (sauf NEXT_PUBLIC_*)
- [ ] Webhooks Stripe vérifiés avec signature
- [ ] Webhooks crypto vérifiés avec secret
- [ ] Logs d'audit : toute action admin est loggée
- [ ] KYC requis au-delà de $500 de retrait
- [ ] 2FA optionnel (fortement encouragé)
- [ ] Session invalidation à chaque changement de mot de passe
- [ ] CORS configuré strictement
- [ ] SQL injection impossible (Supabase parameterized queries)
- [ ] XSS prevention (React escape automatique + CSP headers)
- [ ] CSRF tokens sur formulaires sensibles

## 16. INTERNATIONALISATION DES PAIEMENTS

### Taux de Change
- HTG (Gourde haïtienne) : **Fixe $1 = 140 HTG** (à mettre à jour depuis admin si besoin)
- Autres devises : API Open Exchange Rates (gratuit jusqu'à 1000 req/mois) ou Fixer.io
- Cache 1 heure pour éviter appels excessifs
- Affichage du taux utilisé sur chaque transaction

### Frais de Conversion
- Moncash/Natcash : 0% (frais fixés dans le taux HTG)
- Carte/Stripe : 2.9% + $0.30 (Stripe standard) — afficher clairement avant confirmation
- Crypto : réseau fee variable (affiché en temps réel)
- PayPal : 3.5% international — affiché avant confirmation
- Tous les frais affichés AVANT confirmation, jamais cachés
